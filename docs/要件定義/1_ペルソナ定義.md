# ペルソナ定義

## ペルソナ1：転職を目指す社会人（田中 太郎）

### 基本情報
- **年齢**: 28歳
- **職業**: 営業職（IT企業）
- **居住地**: 東京都
- **プログラミング経験**: 独学で3ヶ月（Python基礎、HTML/CSS基礎）

### 背景・状況
- 現在の営業職に限界を感じ、エンジニアへの転職を検討
- 平日は仕事が忙しく、学習時間は主に平日夜と週末
- オンラインのプログラミング講座を受講中だが、コードレビューを受ける機会が少ない
- 転職活動を半年後に開始予定

### ニーズ・課題
- **コードの品質が正しいか不安**
  - 独学のため、自分のコードが適切か判断できない
  - エラーの原因が分からず、解決に時間がかかる
- **フィードバックが欲しい**
  - コードレビューを受ける機会が少ない
  - 改善点やベストプラクティスを知りたい
- **時間効率の良い学習**
  - 限られた時間で効率的にスキルアップしたい
  - 24時間いつでも質問できる環境が欲しい

### 行動パターン
- 平日：帰宅後（21時〜23時）に学習
- 週末：午前中に集中して学習
- 学習方法：オンライン講座の課題を解き、GitHubにアップロード
- 情報収集：Twitter、Qiita、技術ブログをよく閲覧

### 技術レベル
- **初心者〜初級者**
- 理解していること：変数、関数、基本的な制御構文、簡単なAPI呼び出し
- 苦手なこと：エラーハンドリング、設計パターン、コードの可読性向上

### 使用ツール
- PC（Windows）、VSCode、Git、GitHub
- 学習：Udemy、Progate、ドットインストール

## ペルソナ2：情報系大学生（佐藤 花子）

### 基本情報
- **年齢**: 21歳
- **職業**: 大学3年生（情報工学科）
- **居住地**: 大阪府
- **プログラミング経験**: 大学の授業で2年（Java、C、Python、JavaScript）

### 背景・状況
- 大学の授業でプログラミングを学んでいるが、実践的なコードレビューが少ない
- インターンシップや就活に向けて、より実践的なスキルを身につけたい
- アルバイトをしながら学習しているため、時間に制約がある
- 将来はWebエンジニアを目指している

### ニーズ・課題
- **実践的なコードレビュー**
  - 授業の課題は動けばOKというレベルで、品質まで見てもらえない
  - 企業で求められるコード品質を知りたい
- **就活対策**
  - ポートフォリオのコードを改善したい
  - 面接で説明できるレベルの理解を深めたい
- **学習の継続**
  - モチベーション維持が難しい
  - 定期的なフィードバックで成長を実感したい

### 行動パターン
- 平日：授業の空き時間や放課後に学習
- 週末：アルバイトの合間に学習
- 学習方法：授業の課題、個人プロジェクト、ハッカソン参加
- 情報収集：Zenn、Qiita、技術系YouTube

### 技術レベル
- **初級者〜中級者**
- 理解していること：オブジェクト指向、基本的なアルゴリズム、データベース操作
- 苦手なこと：テストコード、セキュリティ、パフォーマンス最適化

### 使用ツール
- PC（Mac）、VSCode、Git、GitHub、Docker（学習中）
- 学習：大学の教材、Zenn、ハッカソン

## ペルソナ3：副業でエンジニアを目指す会社員（鈴木 一郎）

### 基本情報
- **年齢**: 35歳
- **職業**: 製造業の管理職
- **居住地**: 愛知県
- **プログラミング経験**: 独学で6ヶ月（JavaScript、React基礎）

### 背景・状況
- 本業は安定しているが、副業でエンジニアとして働きたい
- 家族がいるため、学習時間は平日の早朝（5時〜7時）と週末
- オンラインのブートキャンプを受講したが、卒業後の継続的な学習支援が欲しい
- フリーランスとしての活動を1年後に開始予定

### ニーズ・課題
- **実務レベルのコードレビュー**
  - 独学では実務で通用するコードが書けるか不安
  - クライアントに提出するコードの品質を上げたい
- **継続的な学習支援**
  - ブートキャンプ終了後も継続的にフィードバックが欲しい
  - 最新の技術トレンドに対応したい
- **時間効率**
  - 限られた時間で最大限の学習効果を得たい
  - すぐに質問できる環境が必要

### 行動パターン
- 平日：早朝（5時〜7時）に学習
- 週末：午前中に集中して学習
- 学習方法：個人プロジェクト、オンライン講座、技術書
- 情報収集：Twitter、技術ブログ、技術コミュニティ

### 技術レベル
- **初級者**
- 理解していること：JavaScript基礎、React基礎、基本的なAPI開発
- 苦手なこと：複雑な状態管理、パフォーマンス最適化、セキュリティ

### 使用ツール
- PC（Windows）、VSCode、Git、GitHub、Vercel
- 学習：Udemy、技術書、個人プロジェクト

## 共通する特徴

### 共通ニーズ
1. **コードレビューが欲しい**
   - 自分のコードが正しいか、改善できる点があるか知りたい
2. **24時間いつでも質問できる**
   - 学習時間が限られているため、いつでもフィードバックが得られる環境が必要
3. **実践的なフィードバック**
   - 単なるエラー修正だけでなく、ベストプラクティスや改善提案が欲しい
4. **学習の継続性**
   - モチベーション維持のため、定期的なフィードバックで成長を実感したい

### 共通課題
1. **コードレビューを受ける機会が少ない**
   - 独学や大学の授業では、実践的なコードレビューが受けられない
2. **時間的制約**
   - 仕事や学業と両立しながら学習しているため、時間が限られている
3. **フィードバックの質**
   - 単なるエラー修正だけでなく、コードの品質向上につながるフィードバックが欲しい
